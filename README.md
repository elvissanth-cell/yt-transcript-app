# 逐字稿站｜YouTube 搜尋 + 逐字稿 + 留言擷取工具

## 專案結構
```
yt-app/
├── backend/            # 現在同時是 API 也是網站伺服器
│   ├── app.py          # Flask：搜尋/逐字稿/留言 API + 端出前端頁面
│   ├── requirements.txt
│   └── static/          # 前端頁面就放這裡，Flask自動端出
│       ├── index.html
│       ├── manifest.json
│       ├── icon.svg
│       └── sw.js
├── frontend/            # (舊版備份，只想用Netlify分離部署時才用得到)
└── render.yaml
```

## 最簡部署法（推薦）：只需部署一次
現在前端已經整合進 Flask 的 `static/` 資料夾，**只需要部署 `backend` 這一個服務**，
打開它的網址就是完整可用的 App（搜尋、逐字稿、留言、語音輸入全部都在同一個網址上），
不用再另外操作 Netlify Drop，也不用手動改網址對接。

---

## Step 1：本機直接試跑（最快確認能不能動）
```bash
cd backend
pip install -r requirements.txt
python app.py
```
啟動後會看到 `Running on http://127.0.0.1:5001`，保持這個視窗開著。

**注意**：打開瀏覽器要連 `http://127.0.0.1:5001`（不是直接打開 index.html 檔案），
因為現在頁面是由 Flask 端出來的，這樣搜尋/逐字稿/留言才能正常運作。

## Step 2：部署到雲端（只有這一步，一次搞定）
把整個 `yt-app` 資料夾（含 `backend`）推上 GitHub，然後到 Render 建立一個 Web Service：

```bash
cd yt-app
git init && git add . && git commit -m "init"
git remote add origin https://github.com/你的帳號/yt-transcript-app.git
git branch -M main && git push -u origin main
```

1. 前往 https://render.com，用 GitHub 帳號登入（不需信用卡）
2. Dashboard 點 **New → Web Service**，選擇剛剛推上去的 repo
3. Render 會讀到 `render.yaml` 自動帶入設定（若沒自動偵測，手動填）：
   - **Root Directory**：`backend`
   - **Build Command**：`pip install -r requirements.txt`
   - **Start Command**：`gunicorn app:app --bind 0.0.0.0:$PORT`
   - **Plan**：Free
4. 點 **Create Web Service**，等 build 完成
5. 拿到網址，例如 `https://transcript-desk.onrender.com` —— **打開它就是完整的 App**，
   搜尋、逐字稿、留言、語音輸入全部在同一頁，不用再做任何額外設定。

**免費方案限制**：15 分鐘無流量會休眠，下一次請求要等 30–60 秒冷啟動喚醒，
屬正常現象；想全時在線需升級 Starter 方案（約 $7/月）。免費方案本身不需信用卡。

## Step 3：把它「裝」成桌面 App（PWA）
用 Chrome/Edge 打開你的 Render 網址 → 網址列右側會出現「安裝」圖示 →
點擊後就會在桌面/工作列建立一個獨立圖示的 App 視窗，開啟時不會有瀏覽器網址列，
體驗上就像一個桌面應用程式。

## Google 登入 / 我的訂閱動態 —— 需要你申請 OAuth 憑證

這個功能要讀取「你自己的 Google 帳號」的訂閱清單，Google 規定只有帳號本人能申請授權憑證，
我沒辦法幫你申請，以下是完整步驟：

### 1. 建立 Google Cloud 專案
1. 前往 https://console.cloud.google.com/ ，用你的 Google 帳號登入
2. 右上角點「建立專案」，取個名字（例如 `yt-transcript-app`），建立完成

### 2. 啟用 YouTube Data API v3
1. 左側選單 **API 和服務 → 程式庫**
2. 搜尋 `YouTube Data API v3`，點進去，按 **啟用**

### 3. 設定 OAuth 同意畫面
1. 左側選單 **API 和服務 → OAuth 同意畫面**
2. **使用者類型**選「外部」，下一步
3. 填應用程式名稱（隨意，例如「逐字稿站」）、使用者支援電子郵件、開發人員聯絡資訊，儲存
4. **範圍**這步先跳過或不用特別加，儲存並繼續
5. **測試使用者**這步，**把你自己的 Gmail 帳號加進去**（很重要，沒加的話登入會被拒絕），儲存

### 4. 建立 OAuth 用戶端 ID
1. 左側選單 **API 和服務 → 憑證**
2. 點 **建立憑證 → OAuth 用戶端 ID**
3. 應用程式類型選 **網頁應用程式**
4. **已授權的重新導向 URI** 這欄，依你的使用情境加入對應網址：
   - 本機測試：`http://127.0.0.1:5001/auth/callback`
   - 部署到 Render 後：`https://你的服務名稱.onrender.com/auth/callback`
   （兩個都加也可以，之後不管本機或雲端都能用）
5. 點建立，會跳出 **用戶端 ID** 和 **用戶端密鑰**，右上角有個 **下載 JSON** 按鈕，點下去

### 5. 把下載的檔案放進專案
把剛剛下載的 JSON 檔重新命名成 `client_secret.json`，放進 `backend/` 資料夾裡
（跟 `app.py` 同一層）。**這個檔案含有你的密鑰，不要上傳到公開的 GitHub repo**
（我已經在 `.gitignore` 裡把它排除了，正常操作不會不小心傳上去）。

### 6. 測試登入
重新啟動後端（`Ctrl+C` 停掉再 `python3 app.py`），重新整理網頁，右上角應該會出現
**「登入 Google 帳號」**按鈕。點下去會跳轉到 Google 的授權畫面，同意後會跳回你的 App，
按鈕會變成「登出」，同時出現「我的訂閱」按鈕。

### 7. 部署到 Render 之後的差異
若你把 `client_secret.json` 也一起放進 GitHub repo 供 Render 讀取，**強烈不建議**
（密鑰會外洩）。比較安全的做法：改用環境變數，把 JSON 內容整包貼進 Render 的
Environment Variable（之後有需要我可以幫你把程式碼改成優先讀環境變數）。
單純本機使用的話可以先不管這段。

## 這次改版新增了什麼

**三欄式版面**：左側欄（搜尋+篩選+我的最愛置頂，下面是我的訂閱／播放清單／自訂動態導覽，
再下面是篩選出來的影片清單）／中間主欄（播放器＋逐字稿＋留言）／右側欄（開啟播放清單時才出現，
顯示清單內影片，可勾選直接從清單移除）。

**搜尋篩選**：時長／上傳時間／排序，在側邊欄搜尋框下方的三個下拉選單。
＊這些篩選是在抓到的這批搜尋結果內做篩選，不是叫 YouTube 重新用該條件搜尋，條件太窄可能篩到 0 筆。

**播放清單管理**：導覽列「播放清單」會列出你自己帳號建立的清單，點進去可以在右側欄看到內容、
勾選移除、用下拉選單切換清單。

**自訂動態**：導覽列「自訂動態」是關鍵字進階搜尋（例如 `kendo, finance uploaded in past 1 month`），
**不是** YouTube 真實的個人化推薦演算法——那套演算法 Google 從未對外開放 API，這是最接近的替代方案。

**播放速度快捷鍵**：點進影片畫面後，直接用 YouTube 原生快捷鍵 `Shift + .`／`Shift + ,` 加減速，
不需要額外設定。

## 三項確認不做的功能（Google 官方限制，非技術問題）

- **觀看紀錄（History）匯入**：YouTube API 從未開放讀取使用者觀看紀錄，這是隱私限制。
- **稍後觀看（Watch Later）匯入**：Google 在 2016 年起就把這個清單排除在公開 API 之外。
- **真實的「推薦影片／New to you」**：這是 YouTube 內部機器學習系統，沒有對外開放的 API；
  上面的「自訂動態」是唯一能做到的替代方案。

## 在手機上使用

**先搞懂一個常見誤會**：`127.0.0.1` 這個網址，指的永遠是「打開這個網址的那台裝置自己」，
不是「跑後端程式的那台電腦」。所以你在手機瀏覽器打 `http://127.0.0.1:5001`，
手機會去找「手機自己身上」有沒有在跑這個服務——當然沒有，所以顯示連不上。

### 方法一：同一個 WiFi 下，用電腦的區域網路 IP（最快，但只限同網路）
1. 在跑後端的那台電腦上查自己的區網 IP：
   - **Windows**：開命令提示字元，打 `ipconfig`，找「IPv4 位址」，通常長得像 `192.168.x.x`
   - **Mac**：開終端機，打 `ipconfig getifaddr en0`（Wi-Fi）或系統偏好設定→網路
2. 確認手機和電腦連的是**同一個 WiFi**
3. 手機瀏覽器打 `http://你查到的IP:5001`（例如 `http://192.168.1.23:5001`）
4. 如果還是連不上，多半是電腦的防火牆擋掉了區網連線，需要到防火牆設定允許 Python/5001 埠

### 方法二：部署到 Render（最穩定，任何網路都能用）
照上面「Step 2：部署到雲端」把後端放到 Render，拿到 `https://xxx.onrender.com` 這種網址後，
手機直接打這個網址就好，不受 WiFi/IP 限制，這也是長期使用建議的方式。

### Google 登入在區網/手機情境下的注意事項
如果用方法一（區網 IP）登入 Google 帳號，要記得回 Google Cloud Console 的 OAuth 用戶端設定，
把 `http://你的區網IP:5001/auth/callback` 也加進「已授權的重新導向 URI」，不然登入會失敗
（區網 IP 通常不固定，換了路由器或重開機可能會變，這也是方法二更適合長期使用的原因）。

## 「改了東西手機上沒生效」的真正原因(已修正)

這個 App 為了能「加到手機主畫面」用了 Service Worker（`sw.js`），它會把網頁存一份在裝置本機。
早期版本用的是「先讀本機快取,讀不到才連網」的策略，而且快取版本號一直沒變過，
導致手機（尤其是「加到主畫面」過的裝置）可能一直讀著很舊的版本，不管後端改了多少次都沒感覺。

現在已經改成「一律先連網抓最新版，連不到網路才退回本機備份」，並且把快取版本號往上跳一版，
強制清掉舊快取。**但如果你的手機瀏覽器已經卡在舊版本，這次更新後可能還需要手動清一次**：

**iPhone Safari**：設定 App → Safari → 清除瀏覽記錄與網站資料（注意這會清掉其他網站的資料，
如果不想全部清掉，改用下面「Android/桌面」的方法對這個網站單獨處理）

**Android Chrome / 電腦 Chrome**：打開這個網站 → 網址列左邊的鎖頭/資訊圖示 → 網站設定 →
清除資料（或 Cookie 和網站資料）→ 重新整理頁面

清完之後重新打開網址，應該就會抓到最新版本了。之後就不用再手動清了，因為新的快取策略會自動更新。

## 分類/我的最愛現在存在伺服器上（跨裝置同步）

之前分類和我的最愛存在瀏覽器的 localStorage，這是「裝置綁定」的儲存方式——電腦上的 Chrome
和手機上的 Safari 即使打開同一個網址，存的東西完全獨立不互通，這是瀏覽器本身的設計。

現在改成存在後端伺服器上（`backend/categories.json`、`backend/favorites.json`），
不管用哪個裝置打開，看到的都是同一份資料。**第一次升級時，程式會自動把你電腦瀏覽器裡原本的
分類資料搬一次上去**，之後就都是讀伺服器上的版本，不用手動搬移。

**部署到 Render 的注意事項**：Render 的免費方案重新部署或休眠喚醒時，磁碟內容有機會被重置，
`categories.json`/`favorites.json` 可能會不見。如果你長期用 Render 版本，建議偶爾用「管理頻道分類」
把設定截圖備份，或之後有需要我可以幫你改成用 Google Sheets／資料庫存，會更穩定。

## 播放清單批次操作

點開播放清單後，每部影片左邊是**純勾選框**（不會點了就問你要不要移除）。勾選後，
上方工具列會出現：
- **移出此清單**：把勾選的影片從目前清單整批移除
- **移到選定清單**：右邊選單挑一個其他播放清單，按下去會把勾選的影片加進去、同時從目前清單移除（等於「搬移」）
- **清除勾選**

## 批次勾選＋下載／依序播放（適用範圍已擴大）

現在「勾選影片→批次下載逐字稿/留言→依序播放」這套功能，除了頻道頁之外，
**搜尋結果、我的訂閱動態、自訂動態的結果清單也都可以用**，每個影片卡左上角都會有勾選框，
勾選後清單下方會出現工具列（依序播放／下載逐字稿／下載留言／清除勾選）。
下載出來的 txt 檔，每支影片前面都會標註影片標題和連結，方便區分。

依序播放現在共用同一顆右側的主播放器（也就是說也能在播放佇列時用 Shift+. / Shift+, 調速）。

## 我的訂閱／自訂動態現在也能篩選時長／發佈時間

左側欄的時長／發佈時間下拉選單，現在除了搜尋，也會套用在「我的訂閱」動態和「自訂動態」上：
- 我的訂閱：套用篩選時會一併帶入這兩個條件
- 自訂動態：如果你在輸入框裡寫了「uploaded in past N day/week/month/year」，優先用那個；
  沒寫的話會退回用左側欄的發佈時間篩選

## 三欄式版面（最新版）

- **左欄（窄）**：搜尋框＋篩選條件＋我的最愛，下面是導覽按鈕（搜尋結果／我的訂閱／播放清單／自訂動態／YouTube 首頁推薦）
- **中欄**：目前選取的清單內容（搜尋結果、訂閱動態、播放清單內容、自訂動態結果），**點擊影片播放後這欄不會消失**，可以繼續往下選下一部
- **右欄**：影片播放器＋快捷鍵提示＋逐字稿＋留言

點擊「🏠 YouTube 首頁推薦」會在新分頁打開你自己帳號的真實 YouTube 首頁（因為推薦演算法沒有對外 API），
同時站內會顯示一個說明框，可以一鍵跳到「自訂動態」作為站內的替代方案。

## 如果你還是想用 Netlify Drop（選用，非推薦）
`frontend/` 資料夾保留作為備份，若你有特別理由想把前端跟後端拆開部署
（例如前端要放 CDN 加速），可以把 `frontend/index.html` 裡的：
```js
const API_BASE = "";
```

再把 `frontend` 資料夾拖到 https://app.netlify.com/drop 即可。但一般情況下
**不需要這麼做**，Step 2 的單一部署已經足夠。

## 已知風險 / 注意事項
- `yt-dlp`、`youtube-transcript-api`、`youtube-comment-downloader` 都是靠解析
  YouTube 網頁結構運作的非官方工具，YouTube 改版時可能會失效，屆時需更新套件版本。
- 語音輸入的 Web Speech API 在 Chrome/Edge 支援最好，Firefox/Safari 支援度較弱。
- 大量／高頻率請求可能觸發 YouTube 的速率限制，建議個人研究用途酌量使用。
